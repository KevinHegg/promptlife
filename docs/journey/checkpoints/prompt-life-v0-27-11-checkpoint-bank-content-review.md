|---:|---|---|
| 1 | What Is an LLM? | Explain that an LLM uses learned weights plus current context to score and generate next tokens without implying awareness, search, or permanent learning. |
| 2 | Where LLMs Fit | Use the AI family map to distinguish AI, ML, deep learning, generative AI, LLMs, diffusion models, multimodal systems, and AI products. |
| 3 | Rationalists vs Empiricists | Compare rules-first AI and learned-pattern AI while recognizing that modern systems can combine rules, retrieval, policies, tools, and learned models. |
| 4 | Training | Trace training as example, prediction, loss, optimizer, weight update, repeat; separate durable training from inference, retrieval, sampling, and prompt changes. |
| 5 | Pretraining | Explain pretraining as broad durable pattern learning from many next-token targets, not perfect recall, live browsing, or a searchable archive. |
| 6 | Overfitting and Generalization | Diagnose overfitting and explain that set-aside validation examples are kept out of training to test transfer beyond examples used to fit weights. |
| 7 | Fine-Tuning | Distinguish fine-tuning or adapter training as durable post-pretraining adaptation from prompt, RAG, and sampling-time steering. |
| 8 | Alignment | Reason about alignment as behavior shaping through training, prompts, policies, filters, evaluation, and deployment controls without treating it as conscience or truth guarantee. |
| 9 | Inference | Explain inference as a forward pass using fixed weights and temporary context-shaped states, not durable learning. |
| 10 | Prompt vs Response | Separate given prompt/context tokens from generated response tokens and explain that generated tokens are appended into the next inference context. |
| 11 | Tokenization | Explain tokenization as turning text into model-readable chunks that may be words, word pieces, punctuation, or other units before ID lookup. |
| 12 | Token IDs | Explain token IDs as lookup keys from token chunks to embedding rows, not meanings, memories, or source retrieval by themselves. |
| 13 | Embeddings | Distinguish an embedding as a learned starting vector retrieved from a durable embedding table from a temporary hidden state shaped by context. |
| 14 | Vectors | Explain vectors as numerical representations with distributed features and distinguish vectors from scalar IDs, tensors, and human-readable labels. |
| 15 | Tensors | Explain tensors as shaped numerical blocks that organize many token vectors and features for layer operations. |
| 16 | Attention | Explain attention as temporary weighted relevance between token positions that shapes hidden states, not human awareness or final token choice by itself. |
| 17 | MLP | Explain the MLP as per-token feature reshaping inside each layer, distinct from attention mixing across token positions. |
| 18 | Layers | Explain transformer layers as repeated attention-plus-MLP transformations that refine temporary hidden states without implying human thought steps. |
| 19 | Hidden States | Distinguish hidden states as temporary context-shaped internal vectors created during inference from embeddings, weights, visible text, and memory. |
| 20 | Logits | Explain logits as temporary raw next-token scores produced before softmax, not probabilities, truth scores, or complete answers. |
| 21 | Softmax | Explain softmax as converting logits into a next-token probability distribution, not choosing the token or proving truth. |
| 22 | Sampling | Explain sampling as choosing one next token from probabilities during inference, after softmax and before append/repeat. |
| 23 | Autoregression | Explain autoregression as next token, append to context, run again, while weights usually remain unchanged and context grows. |
| 24 | Context Window | Explain the context window as temporary visible input space for the current run, not permanent memory, and show how autoregression and RAG compete for that space. |
| 25 | RAG and Retrieval | Explain RAG as a retrieval system that places outside evidence into current context during inference, not training, omniscience, or permanent memory. |
| 26 | Grounding | Explain grounding as connecting generated claims to available evidence while recognizing that citations or retrieval do not guarantee truth. |
| 27 | Hallucinations | Explain hallucination as fluent generated output that lacks adequate support, not intentional lying, consciousness, or proof of malice. |
| 28 | How AI Learns | Compare ways AI systems appear to learn and separate durable weight/adaptor changes from temporary context use, retrieval, and in-context behavior. |
| 29 | Diffusion vs Autoregression | Distinguish autoregressive text generation from diffusion-style denoising and show that generative AI methods do not all work like ChatGPT. |
| 30 | Multimodal AI | Explain multimodal AI as systems that represent or process more than one media type together without implying human-like perception. |
| 31 | The Perfect Storm | Explain modern LLM capability as convergence among data, compute, storage, methods, labor, and incentives, not one magic breakthrough. |
| 32 | Collective Intelligence, Extracted | Explain that much model usefulness comes from patterns in human-created data and labor, raising questions of provenance, consent, and compensation. |
