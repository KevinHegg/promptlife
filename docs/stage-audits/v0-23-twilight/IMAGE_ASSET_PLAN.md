# Twilight Image Asset Plan v0.23.1

Date: 2026-06-07

## Decision

No generated PNG assets were added in this pass. All four Twilight cards now use coded SVG/HTML visuals so exact labels, accessibility text, and mobile behavior remain under app control.

## Current Visual Strategy

| Card | Current asset approach | Reason |
| --- | --- | --- |
| How AI Learns | Coded learning-modes matrix | The durable-vs-temporary distinction needs exact labels and short matrix cells. |
| Diffusion vs Autoregression | Coded split diagram | The lesson needs a clear append-vs-denoise contrast that would be brittle if baked into a PNG. |
| Multimodal AI | Coded media lane map | The visual must show media inputs, a bounded shared space, and outputs without implying human perception. |
| The Perfect Storm | Coded convergence streams | The current pass needed ingredient clarity before atmosphere. |

## Future Image 2 Candidate

The Perfect Storm is the best future Image 2 candidate for Twilight.

Recommended direction:

- Textless ZenTron Origami convergence scene.
- Data pages, compute glow, storage/checkpoint forms, folded method paths, human-evaluation markers, and incentive currents converging toward a model-capability cloud.
- Keep labels, captions, alt text, key takeaways, and instructional copy in HTML.
- Use only after the coded ingredient interaction remains stable through QA.

## Not Recommended For PNG In This Pass

- How AI Learns: exact distinctions matter too much.
- Diffusion vs Autoregression: the split comparison needs precise labels.
- Multimodal AI: the map needs to stay inspectable and accessible.
