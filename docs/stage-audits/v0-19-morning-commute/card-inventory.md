## 9. Inference

- Subtitle: A forward pass, not a memory update.
- Path type: Essential.
- Terms: inference, forward-pass, input-context, weight, hidden state, logits, sampling.
- One-sentence definition: Inference is normal model use: fixed learned weights process the current context to produce scores for the next response token.
- Core explanation: During ordinary use, the current context enters the model, embeddings and hidden states are computed, the final hidden state produces next-token scores, and decoding chooses one response token. Then the token is appended and the process repeats. The learned weights are used, but they normally stay fixed.
- Where it happens: Every time a deployed model answers a prompt.
- Durable or temporary: Inference creates temporary hidden states and next-token scores for this run. It normally does not durably change the model weights.
- Prompt vs response note: Prompt tokens, retrieved context, conversation history, and response-so-far tokens can all be current context; inference generates the next response token.
- Why it matters: This is the line between using a trained model and training a model.
- Relationship: Batch 1 explained how weights are shaped before use. Inference is the live run that uses those weights without rewriting them.
- Metaphor: Using a map to choose a route, not redrawing the map.
- Brain Bridge: Like using working memory to answer from the information currently in front of you. Limit: the model is not conscious or living; it computes temporary vectors with fixed weights.
- Misconception corrected: Normal chat automatically trains the model.
- Visual aid: `inference-pass`, a coded forward-pass pipeline with context, temporary states, fixed weights, and next-token scores.
- Tiny interaction: `inference`, "Walk the forward pass."
- Checkpoint: What changes during ordinary inference? Correct answer: Temporary hidden states.
- Current assessment: Accurate and well placed. The durable/temporary distinction is strong. The next pass should make the forward pass more tangible by showing fixed weights below temporary activation states and one score set at the end.

## 10. Prompt vs Response

- Subtitle: Given context versus generated tokens.
- Path type: Essential.
- Terms: prompt, response, prompt-tokens, response-tokens, generated-token, input-context, completion.
- One-sentence definition: Prompt tokens are given to the model; response tokens are generated by the model one at a time and appended to the context.
- Core explanation: The user prompt should be a complete request, such as: "Give me one sentence with two household pets in conflict." The model then begins generating a response. During generation, the response-so-far may be incomplete: "A jealous dog chased a startled cat across the kitchen". After the model chooses one response token, such as "floor", that token is appended. The next forward pass sees the user prompt plus the response so far.
- Where it happens: During inference and response generation.
- Durable or temporary: Prompt and response-so-far tokens are temporary context. They can influence the current run, but they do not normally update weights.
- Prompt vs response note: This is the distinction itself: the user prompt is given context; "floor" is a generated response token appended to the response-so-far.
- Why it matters: It prevents the myth that the model writes a full answer all at once.
- Relationship: After we distinguish prompt from response, we can see how both are represented as tokens.
- Metaphor: Given cards on a table, then new cards added one at a time.
- Brain Bridge: Like hearing a sentence and continuing it word by word. Limit: a person may plan, intend, and revise; the model repeatedly samples next tokens from probabilities.
- Misconception corrected: The model generates the whole response at once.
- Visual aid: `prompt-response`, a coded prompt/response/context diagram.
- Tiny interaction: `autoregressive`, "Append one token."
- Checkpoint: What happens after one response token is generated? Correct answer: It is appended to the context.
- Current assessment: Strong, distinct, and necessary. It should remain separate from the later Autoregression card: this card defines roles in the current context, while Autoregression should teach the repeated loop.

## 11. Tokenization

- Subtitle: Text becomes model-readable chunks.
- Path type: Essential.
- Terms: tokenization, tokenizer, token, prompt-tokens, response-tokens, token-id, embedding.
- One-sentence definition: Tokenization splits text into chunks that can be represented as token IDs.
- Core explanation: LLMs do not carry raw English sentences through their layers. Text is first split into tokens: pieces that might be whole words, word parts, punctuation, or spaces. The generated response "A jealous dog chased a startled cat across the kitchen floor." can be shown as simplified token cards for learning. Real tokenizers may split it differently. Prompt text is also tokenized before model processing, and generated response tokens are token IDs before they are displayed as text.
- Where it happens: Before embedding lookup and before transformer layers process the current context.
- Durable or temporary: Tokenization itself does not change model weights. It is a representation step during input/output processing.
- Prompt vs response note: Both prompt text and generated response text are represented as tokens. Generated response tokens are chosen as token IDs, then displayed as text.
- Why it matters: Without tokenization, the model has no numerical pieces to process.
- Relationship: Tokens become token IDs, and token IDs retrieve embeddings.
- Metaphor: Parcels on a conveyor belt entering the model.
- Brain Bridge: Like hearing a sentence in meaningful chunks before interpreting it. Limit: human language perception uses sound, meaning, memory, and experience; tokenization uses tokenizer rules and token IDs.
- Misconception corrected: Tokens are always whole words or human concepts.
- Visual aid: `tokenization`, a coded token-chunk diagram.
- Tiny interaction: `tokens`, "Tap to split the response."
- Checkpoint: What can a token be? Correct answer: A word, word piece, punctuation mark, or other text chunk.
- Current assessment: Clear and appropriately placed. The visual and tiny interaction should more explicitly show uneven chunks, punctuation, spaces, and word pieces so learners do not silently re-import the "tokens are words" myth.

## 12. Token IDs

- Subtitle: Chunks become lookup numbers.
- Path type: Essential.
- Terms: token-id, token, tokenizer, embedding-table, embedding.
- One-sentence definition: A token ID is the number assigned to a token so the model can look up its learned starting vector.
- Core explanation: After tokenization, each token is represented by an integer ID. The ID is not the meaning. It is a lookup key. For this app example, dog maps to 421, cat maps to 982, and floor maps to 1576. The embedding table uses each ID to retrieve a learned starting vector. During generation, the model also chooses a next token ID before it is displayed as text.
- Where it happens: Between tokenization and embedding lookup.
- Durable or temporary: Token IDs are part of representation during inference. The mapping is fixed by the tokenizer/model setup; using an ID does not update weights.
- Prompt vs response note: Prompt tokens become token IDs before processing. Generated response tokens are also token IDs before being shown as text and appended to context.
- Why it matters: Token IDs are the bridge from text chunks to numerical computation.
- Relationship: Token IDs point to rows in the embedding table.
- Metaphor: A library call number.
- Brain Bridge: A little like recognizing a symbol and activating a familiar association. Limit: humans do not assign fixed integer IDs to every language chunk; the number itself is not understanding.
- Misconception corrected: The token ID is the meaning.
- Visual aid: `token-ids`, a coded token-to-ID-to-table diagram.
- Tiny interaction: `tokens`, "Match chunks to IDs."
- Checkpoint: Why does the model need token IDs? Correct answer: To look up embeddings.
- Current assessment: Accurate, but the visual should separate the tokenizer, the ID mapping, and the embedding table as three distinct pieces. The current tiny interaction reuses the token interaction pattern and could become more diagnostic.

## 13. Embeddings

- Subtitle: Token IDs look up learned starting vectors.
- Path type: Essential.
- Terms: embedding, embedding-table, token-id, vector, hidden state, inference, training.
- One-sentence definition: An embedding is a learned starting vector retrieved for a token ID.
- Core explanation: The embedding table is learned during training. At inference time, each token ID in the current context retrieves one row from that table: for example, floor -> 1576 -> one starting vector. That row is the token's starting vector before context changes it. Later layers turn embeddings into hidden states shaped by surrounding tokens.
- Where it happens: At the start of model processing, before transformer layers.
- Durable or temporary: The embedding table is durable learned model data. The retrieved embedding vector is used temporarily during a forward pass.
- Prompt vs response note: Prompt tokens and response-so-far tokens are looked up as embeddings when they are inside the current context. A newly generated token will get its embedding during the next forward pass after it is appended.
- Why it matters: Embeddings are the first step from token IDs into the model's learned numerical space.
- Relationship: Embeddings are vectors. The next lesson explains vectors more generally.
- Metaphor: A token gets a starting address in a giant meaning cloud.
- Brain Bridge: Like the first associations that light up when you hear a word. Limit: a human association includes memory, sensation, and experience; an embedding is a learned row of numbers.
- Misconception corrected: An embedding is a dictionary definition or a human memory.
- Visual aid: `embeddings`, a coded embedding lookup diagram.
- Tiny interaction: `embeddings`, "Look up a starting vector."
- Checkpoint: What is an embedding in this app? Correct answer: A learned starting vector for a token ID.
- Current assessment: Strong conceptual copy. The next pass should dramatize the contrast between the durable embedding table and the temporary retrieved vector. This is the strongest candidate for a textless Image 2 asset if only one Morning Commute generated asset is made.

## 14. Vectors

- Subtitle: Lists of numbers carrying features.
- Path type: Essential.
- Terms: vector, feature, distributed-representation, embedding, hidden state, activation.
- One-sentence definition: A vector is a list of numbers that represents features of something, such as a token.
- Core explanation: In an LLM, many things are represented as vectors. An embedding is a starting vector for a token ID. A hidden state is a later vector shaped by context. The numbers are not usually neat labeled features like animal-ish, grammar role, tone, position, or context clue. Those labels are teaching simplifications. Real features are distributed across many dimensions, which is why vectors can represent fuzzy relationships.
- Where it happens: Throughout the model: embeddings, hidden states, attention calculations, MLP transformations, and output scoring all involve vectors.
- Durable or temporary: Some vectors are durable learned parameters, such as embedding rows. Many vectors during inference are temporary activations, such as hidden states.
- Prompt vs response note: Prompt tokens and response-so-far tokens become vectors when processed in the current context. Those vectors help choose the next response token, such as "floor".
- Why it matters: Vectors let the model compute with many fuzzy features at once instead of using plain-English labels.
- Relationship: Several token vectors organized together become a tensor.
- Metaphor: A dashboard of many sliders.
- Brain Bridge: Like many weak associations around a word lighting up together. Limit: the model's vector dimensions are learned numerical patterns, not conscious associations or human-labeled ideas.
- Misconception corrected: Each vector dimension has a simple human-readable label.
- Visual aid: `vectors`, a coded feature-bar diagram.
- Tiny interaction: `embeddings`, "Read the vector bars."
- Checkpoint: Which statement is most accurate? Correct answer: Embeddings and hidden states are both vectors.
- Current assessment: The content is accurate, but this is one of the thinnest cards because vectors are abstract and the interaction is not tightly matched. Keep the "labels are simplified" warning, but make the learner manipulate a simplified vector while repeatedly naming the limitation.

## 15. Tensors

- Subtitle: Organized blocks of vectors.
- Path type: Essential.
- Terms: tensor, vector, activation, weight-tensor, input-context, layer.
- One-sentence definition: A tensor is an organized block of numbers, often holding many token vectors at once.
- Core explanation: Once tokens have embeddings, the model needs to carry many token positions and many features together. A simple way to picture this is a table: token positions down one side, feature numbers across the other. With batches, heads, and layers, the shapes get richer. But the core idea is simple: tensors organize vectors so transformer layers can process the current context.
- Where it happens: Tensors carry data through the transformer layers during a forward pass. Model weights are also stored as tensors.
- Durable or temporary: Weight tensors are durable learned parameters. Activation tensors during inference are temporary.
- Prompt vs response note: The current context tensor may include the complete user prompt, retrieved tokens, prior conversation tokens, and response-so-far tokens such as "A jealous dog chased a startled cat across the kitchen".
- Why it matters: The transformer transforms tensors, not raw words.
- Relationship: Now that current context is in tensor form, transformer layers can apply attention and MLPs. That leads into Workday.
- Metaphor: A stack of spreadsheets carrying token features through the model.
- Brain Bridge: Like keeping many signals active at once. Limit: the brain is not literally storing rectangular arrays with token and feature axes.
- Misconception corrected: A tensor is one word, one number, or a mysterious object.
- Visual aid: `tensors`, a coded tensor block diagram.
- Tiny interaction: `tensor`, "Stack the vectors."
- Checkpoint: Which is the better definition of tensor here? Correct answer: An organized block of numbers.
- Current assessment: Accurate and readable at 320px, but it needs the strongest visual support. This is the best CSS 3D candidate: a rotatable token-position-by-feature object with a simple shape note, not a heavy 3D library.
