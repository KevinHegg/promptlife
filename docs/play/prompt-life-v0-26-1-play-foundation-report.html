Prompt Life v0.26.1

Play Challenge Foundation Report

A small foundation pass for calm, replayable Play challenges: shared model, shared storage, shared UI chassis, final slate registry, and mobile QA.

typecheck passed build passed build:pages passed audit:answers passed 320px QA 390px QA

1. Summary

Play now has a consistent foundation for five final challenge slots: Glossary Dojo, Context Stack, Probability Picker, Prompt Run, and Attention Match. The pass tracks practice progress without scores, leaderboards, streaks, or punitive states.

  • Added shared Play TypeScript types for challenge metadata, attempts, outcomes, and progress.
  • Added versioned localStorage helpers under promptlife.playChallenges.v1.
  • Added reusable Play UI components and the ZenTron Paper Lab style layer.
  • Replaced the old Play page sections with the final challenge slate.
  • Connected Glossary Dojo, Context Stack, Prompt Run, and Attention Match to shared Play progress while preserving their existing progress systems.
  • Moved Token Pipeline Relay and How AI Learns out of the normal final slate while keeping compatibility metadata.
  • Updated Badge's Play stat to show final-slate challenge completion slots, without changing badge unlock rules.

2. Recommendations For The Next Pass

  • Build Probability Picker next, focused on logits, softmax, sampling, and probability-is-not-truth feedback.
  • Turn Attention Match into a clearer replayable matching challenge rather than the current Attention Weave bridge.
  • Refactor the live Context Stack screen onto the new PlayChallengeShell and board primitives.
  • Add a small debug-only progress bridge panel if legacy/local Play state needs to be inspected.

3. Challenges And Risks

  • Legacy Play progress is split across multiple keys, so this pass bridges old progress rather than migrating or deleting it.
  • Glossary Dojo still owns its mature mastery storage. The shared Play key records starts and completions only.
  • Probability Picker is intentionally disabled as foundation-ready, so there is no fake completion state.
  • Attention Match still uses the existing Attention Weave interaction under a clearer final-slate name.
  • The existing Vite large-chunk warning remains.
