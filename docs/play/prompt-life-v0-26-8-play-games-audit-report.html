Prompt Life v0.26.8 audit

Play Games UX + Mechanics Audit

Current-state audit of the five visible Play challenges: Glossary Dojo, Attention Match, Probability Picker, Context Stack, and Prompt Run. This pass did not implement gameplay changes.

Executive Findings

Strongest Today

Glossary Dojo is the most complete loop. Context Stack is the clearest small model-mechanics game.

Needs A Build Pass

Probability Picker is still a placeholder, and Attention Match still uses the old Attention Weave mechanic.

Capstone Risk

Prompt Run remains instructionally essential, but it still feels closer to a long guided worksheet than a replayable challenge.

Mobile Result

390px and 320px Play landing checks showed no horizontal overflow. Cards are tall at 320px, but vertical scrolling is obvious.

Scope

Reviewed src/features/play, the challenge registry, shared Play components, shared storage, Glossary Dojo integration, Context Stack, Attention Match, Probability Picker, Prompt Run, Badge language, reset behavior, and Play styles. Screenshots were captured from the local app at http://127.0.0.1:5173/?debug=1&v=0268-play-audit.

No Journey order, badge logic, checkpoint randomization, dependencies, generated assets, or core game mechanics were changed in this pass.

Game-By-Game Notes

Glossary Dojo - Keep

Dojo has the cleanest learning loop: real glossary terms, close distractors, explicit feedback, repeat/review modes, and independent mastery. It preserves old Dojo progress while also writing shared Play summary progress. Mobile risk is mainly vertical: answer choices can run under the bottom nav until the learner scrolls, but controls remain usable.

Attention Match - Rebuild

The concept is right, but the current action is one legacy Attention Weave example. It should become a small set of sentence cards: choose a target token, connect it to the most relevant prior token, then explain that attention is relevance between positions, not awareness.

Probability Picker - Build

This is the most important empty slot. It should teach logits, softmax, probabilities, sampling, uncertainty, and why likely does not mean true. The current disabled card is coherent and does not create fake progress.

Context Stack - Keep

Context Stack now has a strong v2 loop: place cards, see overflow, keep key context visible, and use retrieved evidence only after it enters the window. It already uses the shared Play shell and storage helpers. Future polish should focus on shorter card labels at 320px and slightly more varied replay.

Prompt Run - Revise

Prompt Run carries the whole "day in the life of a prompt" story and badge relevance. Its risk is experience shape: 13 steps plus exercises can feel like a long lesson path. A v2 should make the prompt visibly move through fewer stronger scenes, while keeping the same core distinctions.

Recommended Build Order

  1. Probability Picker v1. Fill the empty slate slot and teach the Decision Room mechanics that no current Play game practices directly.
  2. Attention Match v1. Replace the compatibility bridge with a purpose-built shared-shell challenge.
  3. Prompt Run v2. Keep the capstone but shorten the feel, add stronger continuity, and align it with shared Play language.

Context Stack no longer needs to be first because its v2 refactor is already in place. A later Context Stack v2.1 pass can tune labels, replay, and visual density.

Unified Play Interaction Language

Pattern Recommendation
Action verbs Name, connect, choose, place, trace. Show these as practice moves, but keep the sentence after them concrete.
Status pills Ready, In progress, Review suggested, Completed, Coming soon, Retired. Avoid score-like status labels.
Stat row Played, Current, Progress saved, Review suggested, Completed. Replace future "Best" wording with "Progress" or "Last round" where possible.
Board zones Use consistent zones: incoming/source, working/choice area, current tray/window, fell-out/history, feedback, reflection.
Cards/chips/trays Cards are movable ideas; chips are terms or tokens; trays are temporary holding areas; rails show progress through a round, not performance.
Feedback panels Correct: "Good distinction" or "Insight strengthened." Review: "This choice reveals a common mix-up." Completion: "Insight unlocked" or "Round completed."
Storage Record attempts on meaningful start, update progress during rounds, complete only when an insight or round finishes, and preserve legacy keys by bridging summaries.

Learner-Facing Copy Issues Found

  • Best: Not yet, Best result, and Best: N of 12 are mild score-like remnants. Prefer Progress, Last round, or Current.
  • Back to games appears on Prompt Run and Attention Match while the landing now says Play challenges. Prefer Back to Play.
  • Attention Match still says "source token" and "target token" before learners have much help in that screen. A rebuild should add a one-line setup.
  • Probability Picker has an internal coming-soon screen, but the visible disabled card is the only reachable placeholder from the normal slate.
  • Badge stats still say 0 of 5 play challenges. The criterion panel clarifies Play is practice history, but a future pass could make the stat label even less requirement-like.
  • No punitive language, leaderboards, timers, streaks, or score emphasis were found in the normal Play surface.
